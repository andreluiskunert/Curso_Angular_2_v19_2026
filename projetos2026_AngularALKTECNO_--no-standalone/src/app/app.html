Módulos em Angular

Resumo da Aula – Angular: Módulos e Lazy Loading

Nesta aula, foi apresentado como criar o primeiro módulo no Angular utilizando o Angular CLI. O objetivo principal foi organizar melhor a aplicação, separando funcionalidades em módulos específicos.

Criação do módulo

Foi utilizado o comando ng g m para gerar um módulo. Além disso, foi usada a flag --routing, que também cria um arquivo de rotas para esse módulo.

Exemplo de comando:

ng g m features/produto --routing

Com isso, o Angular cria dois arquivos principais:

  • O arquivo do módulo
  • O arquivo de rotas do módulo

Organização por funcionalidade

O módulo criado foi o de produto, dentro da pasta features. Essa abordagem ajuda a organizar melhor a aplicação por funcionalidades.

Criação dos componentes

Dentro do módulo de produto, foram criados dois componentes:

  • Componente de cadastro
  • Componente de listagem

Exemplo de comandos:

ng g c features/produto/cadastro
ng g c features/produto/listagem

Diferente do que acontecia antes, agora esses componentes não ficam diretamente no módulo principal da aplicação, mas sim dentro do módulo da funcionalidade.

Lazy Loading

A aula também apresentou o conceito de Lazy Loading, que significa carregamento sob demanda. Isso quer dizer que o módulo de produtos só será carregado quando a rota de produtos for acessada.

Exemplo:

{
  path: 'produto',
  loadChildren: () => import('./features/produto/produto.module')
    .then(m => m.ProdutoModule)
}

Essa técnica melhora a organização e também o desempenho da aplicação, pois evita carregar tudo de uma vez logo no início.

Rotas internas do módulo

No arquivo de rotas do módulo de produto, foram configuradas rotas internas para:

  • Listagem de produtos
  • Cadastro de novo produto

A rota padrão foi definida para abrir primeiro a listagem de produtos.

Resultado da aplicação

Ao final, a aplicação passou a ter navegação entre páginas como:

  • Home
  • Novo
  • Produtos

Quando o usuário clica em Produtos, o Angular carrega o módulo de produto e exibe inicialmente a tela de listagem.

Conclusão

A aula mostrou como criar módulos no Angular, organizar a aplicação por funcionalidades, gerar componentes dentro desses módulos e configurar o Lazy Loading para carregar recursos apenas quando necessário.
