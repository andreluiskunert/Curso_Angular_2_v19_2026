Tipos de data bindings

{{frase}}

Nesta aula, foi apresentado o conceito de Data Binding no Angular, que representa a forma como os dados são conectados entre o componente (TypeScript) e o template (HTML). Esse mecanismo permite exibir informações na tela e também capturar ações do usuário, tornando a aplicação dinâmica e interativa. O Angular possui quatro tipos principais de data binding, cada um com uma finalidade específica dentro do desenvolvimento de interfaces.

O primeiro tipo é a interpolação, que utiliza chaves duplas {{frase}} para exibir valores de variáveis diretamente no HTML. Já o property binding permite passar dados do componente para propriedades de elementos HTML, como no caso de imagens usando o atributo src. Esses dois tipos são fundamentais para apresentar dados de forma dinâmica na interface do usuário.

O terceiro tipo é o event binding, que possibilita capturar eventos do usuário, como cliques e teclas pressionadas, executando funções definidas no componente. Ele utiliza parênteses, como (click). Esse recurso é essencial para tornar a aplicação interativa, permitindo respostas imediatas às ações do usuário.

Por fim, o two-way data binding combina entrada e saída de dados simultaneamente, utilizando [(ngModel)]. Com isso, qualquer alteração no input do usuário é refletida automaticamente na variável do componente e vice-versa. Para funcionar corretamente, é necessário importar o módulo de formulários do Angular. Esse tipo de binding é muito utilizado em formulários e aplicações que exigem atualização em tempo real.
